### 2. Settings Profile — Hardcoded Email
**File**: `apps/web/src/app/(dashboard)/settings/page.tsx` (line 411-412)

Shows hardcoded `operator@influenceai.dev` instead of the actual logged-in user.

**Fix**: Fetch user from Supabase auth session (same pattern as sidebar).

Current pipeline-generated content has non-AI titles like "Chimpanzees in Uganda" and "WireGuard Windows release" because the signal sources (RSS feeds, HackerNews) don't filter for AI relevance.

**Fix**: This is a pipeline filtering issue, not a UI issue. Noted for Phase 4. The UI should display whatever data exists faithfully.

## Changes

### 1. Replace GitHub Trends Detail with Generic Pipeline Detail

**Delete**: `apps/web/src/app/(dashboard)/pipelines/github-trends/page.tsx`

**Create**: `apps/web/src/app/(dashboard)/pipelines/[slug]/page.tsx`

This page will:
- Accept any pipeline slug (`github-trends`, `signal-amplifier`, `release-radar`)
- Query `pipeline_runs` for this pipeline's run history
- Query `content_items` for content generated by this pipeline
- Show pipeline definition from the registry (name, description, schedule)
- Show empty states when no runs exist

**Sections**:
1. **Header**: Pipeline name + description from registry
2. **Stats row**: Total runs, success rate, last run time (from `pipeline_runs`)
3. **Recent runs table**: Status, started_at, signals ingested, items generated, duration
4. **Generated content list**: Recent content_items for this pipeline

**No configuration panel** — that's Phase 4 work. Don't show UI for features that don't exist yet.

### 2. Update Pipelines List Page Links

**Modify**: `apps/web/src/app/(dashboard)/pipelines/page.tsx`

Change the "View Details" link from `/pipelines/github-trends` to `/pipelines/[slug]` for each pipeline card.

### 3. Fix Settings Profile

**Modify**: `apps/web/src/app/(dashboard)/settings/page.tsx`

Replace hardcoded profile with Supabase auth user data. Use `createClient()` server-side to get the current user's email.

### 4. Add Pipeline Detail Query Functions

**Create**: Add to `apps/web/src/lib/queries/pipelines.ts`:
- `getPipelineRuns(slug, limit)` — recent runs for a specific pipeline
- `getPipelineContentItems(slug, limit)` — content items for a specific pipeline
